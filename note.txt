


After starting the server:
When I start my backend using `node index.js`, first the entry point file runs.
 Inside index.js, I connect to the database using connectDB(). 
 If the database connection is successful, then I start the server using app.listen(). 
 At that point, the server is ready and starts waiting for incoming requests.


After that, everything is handled by app.js. In app.js, I create the express app, 
configure middlewares like JSON parsing and cookies, 
and then mount all routes using app.use("/api/v1", routes).
So whenever a request comes, app.js decides which route file should handle it based on the URL.
then its redirect to that specific routes/xx.js

healthCheck working :
if someone hits `/api/v1/healthcheck`, the request first goes to app.js, 
and from there it gets redirected to route/health.route.js.

In health.route.js, I define the route like this:

"router.get("/healthcheck", verifyJWT, verifyRole("admin"), healthCheck)"
                          --middleware--,--middleware-- ,--controller--

So when this endpoint is called, the request does not directly go to the controller. 
It first passes through authentication and authorization middlewares.

First middleware (authentication), "verifyJWT" runs. In this middleware, 
I extract the token either from cookies (accessToken) or from the Authorization header 'Bearer'. 
Then I verify the token using jwt.verify().
**The verification use the accessToken_secret key which is i used when making the access token
After verification we decode  the token,  
and get the 'user id' that was stored inside it.
**The user id is preset in payload we make the token payload using the userid ans user role while creating token 
Using that id, I find the user in the database. 
If the user exists and the token is valid, 
then we attach the user object which fetch by DB to req.user and call next(). 
This step confirms who the user is. This is Authentication.

After that the next middleware is activates , verifyRole("admin") runs. 
Here I check if the the user obj we get from the prev middleware is contain the  same role 
as we allow to this middleware function i,e 'admin ; then 
If the user is allowed, I call next().

Finally, the request reaches the healthCheck.controller. Here the actual business logic is written. 
It performs the required operation (like returning system status) and sends the final response to the client.

So the complete flow becomes:

index.js → app.js → route → verifyJWT → verifyRole → controller → response

Authentication means "Who are you?"
Authorization means "Are you allowed to do this?"
Controller means "Now perform the actual work."

user registration process :

When user sends a POST request to register, 
first the request goes to app.js then its redirect it to auth.route.js from there the register post trigger 
then  validation middleware stat execute. 
i.e in first  The "userRegisterValidator" checks the fields like email, username and password using express-validator. 
It automatically reads the data from req.body. 
If something is wrong, the validate middleware collects the errors and throws an ApiError. 
Express then sends the error response using global error handler. 
So the controller will not run if validation fails.

If validation passes, then the request goes to the registerUser controller. 
First, I extract email, username and password from req.body. 
Then I check in database if a user already exists with same email or username using $or query. 
If user exists, I throw a 409 conflict error.

If user does not exist, I create a new user with isEmailVerified set to false. 
After that, I generate an email verification token using a method from the User model. 
That method gives me an unhashed token (to send to user), a hashed token (to store in database), 
and an expiry time.

I store the hashed token and expiry in the database and save the user. 
Then I send the unhashed token in a verification link to the user’s email using mailgen.

Finally,I fetch the created user again but remove sensitive fields like password, refreshToken and verification tokens. 
Then I send a success response with status 201 saying user registered successfully and verification email has been sent.
This is my complete registration flow till now.
